## 一、产品概述

### 1.1 产品定位
基金管家是一款面向个人投资者的基金管理工具,提供实时基金行情监控、持仓管理、收益分析等功能,帮助用户更好地管理和跟踪自己的基金投资组合。

### 1.2 产品目标
- 为用户提供实时、准确的基金估值和涨跌信息
- 帮助用户便捷地管理个人基金持仓
- 提供清晰的收益数据可视化,辅助投资决策

### 1.3 目标用户画像
- **主要用户**: 持有多只基金的个人投资者
- **年龄段**: 25-45岁
- **投资经验**: 初级到中级投资者
- **核心需求**: 实时了解基金涨跌、管理持仓、计算收益

## 二、核心功能需求

### 2.1 实时行情显示模块

#### 2.1.1 功能描述
用户可以查看关注基金的实时估值和涨跌情况。

#### 2.1.2 功能要点
- **实时估值显示**
  - 显示基金当前估值净值
  - 显示估值时间(交易日实时更新)
  - 区分估值与实际净值
  
- **涨跌幅展示**
  - 当日涨跌幅(百分比)
  - 当日涨跌额(金额)
  - 涨跌颜色标识(红涨绿跌或绿涨红跌,可配置)

- **基金基本信息**
  - 基金代码
  - 基金名称
  - 基金类型(股票型/混合型/债券型等)
  - 基金公司
  - 基金经理

#### 2.1.3 数据更新机制
- 交易日: 9:30-15:00 每15分钟更新一次估值
- 非交易日: 显示上一交易日净值
- 支持手动刷新

#### 2.1.4 界面要求
```

│ 基金代码: 000001                     │
│ 基金名称: 华夏成长混合                │
├─────────────────────────────────────┤

│ 估值时间: 2026-02-02 14:45           │
│                                     │
│ 涨跌幅: +2.34% ↑                    │
│ 涨跌额: +0.0538                     │
├─────────────────────────────────────┤
│ 昨日净值: ¥2.2918                    │
│ 基金经理: 张三                       │
└─────────────────────────────────────┘
```

### 2.2 持仓管理模块

#### 2.2.1 功能描述
用户可以添加、编辑、删除自己持有的基金,并查看持仓详情和收益情况。

#### 2.2.2 功能要点

**2.2.2.1 添加持仓**
- 输入基金代码或搜索基金名称
- 录入持仓信息:
  - 持有份额
  - 成本净值(平均买入价)
  - 买入日期
  - 备注(可选)

**2.2.2.2 持仓列表展示**
- 显示所有持仓基金
- 每只基金显示:
  - 基金名称和代码
  - 持有份额
  - 成本净值
  - 当前净值(估值)
  - 持仓市值
  - 持仓收益(金额)
  - 持仓收益率(百分比)
  - 今日收益

**2.2.2.3 持仓编辑**
- 修改持有份额
- 修改成本净值
- 追加买入(自动计算新的平均成本)
- 部分卖出(自动调整份额)

**2.2.2.4 持仓删除**
- 支持单个删除
- 删除前二次确认

**2.2.2.5 持仓统计**
- 总资产
- 总成本
- 总收益
- 总收益率
- 今日收益

#### 2.2.3 数据字段定义

| 字段名称 | 字段说明 | 计算公式 |
|---------|---------|---------|
| 持有份额 | 用户持有的基金份额 | 用户输入 |
| 成本净值 | 平均买入价格 | 用户输入或自动计算 |
| 当前净值 | 最新估值净值 | 实时获取 |
| 持仓成本 | 总投入金额 | 持有份额 × 成本净值 |
| 持仓市值 | 当前总价值 | 持有份额 × 当前净值 |
| 持仓收益 | 盈亏金额 | 持仓市值 - 持仓成本 |
| 持仓收益率 | 盈亏百分比 | (持仓收益 / 持仓成本) × 100% |
| 今日收益 | 当日盈亏 | 持有份额 × (当前净值 - 昨日净值) |

## 三、非功能性需求

### 3.1 性能要求
- 页面加载时间 < 2秒
- 数据刷新响应时间 < 1秒
- 支持至少50只基金同时监控

### 3.2 数据准确性
- 估值数据来源于可靠的第三方数据接口
- 明确标识"估值"与"实际净值"的区别
- 提供数据更新时间戳

### 3.3 数据安全
- 用户持仓数据本地存储
- 支持数据导出备份
- 支持数据导入恢复

### 3.4 兼容性
- 支持主流浏览器(Chrome, Safari, Firefox, Edge)
- 响应式设计,支持移动端访问

## 四、技术方案建议

### 4.1 数据来源
- **天天基金网 API**: 实时估值数据
- **东方财富网 API**: 基金基本信息
- **备选方案**: 蚂蚁财富、新浪财经等公开接口

### 4.2 技术栈建议
- **前端**: React/Vue.js + TypeScript
- **状态管理**: Redux/Pinia
- **数据存储**: LocalStorage / IndexedDB
- **图表库**: ECharts / Chart.js
- **UI框架**: Ant Design / Element Plus

## 六、用户故事

### 6.1 实时监控场景
> 作为一名基金投资者,我希望能够实时查看我关注的基金涨跌情况,以便及时了解市场动态。

**验收标准**:
- 能够看到基金当前估值
- 能够看到涨跌幅和涨跌额
- 数据更新及时(交易时间内15分钟内)

### 6.2 持仓管理场景
> 作为一名持有多只基金的投资者,我希望能够集中管理我的所有持仓,以便清晰了解我的投资收益情况。

**验收标准**:
- 能够添加我持有的基金
- 能够看到每只基金的收益情况
- 能够看到总体收益统计

### 6.3 收益计算场景
> 作为一名投资者,我希望系统能够自动计算我的持仓收益,以便我不需要手动计算复杂的收益数据。

**验收标准**:
- 收益计算准确
- 显示持仓收益和今日收益
- 显示收益率百分比

## 七、风险与限制

### 7.1 数据风险
- **风险**: 第三方API可能不稳定或停止服务
- **应对**: 准备多个数据源备选方案,实现数据源切换机制

### 7.2 估值准确性
- **限制**: 估值数据仅供参考,与实际净值可能存在偏差
- **应对**: 明确标注"估值"字样,提示用户以基金公司公布的净值为准

### 7.3 法律合规
- **风险**: 不得提供投资建议,避免违反金融监管规定
- **应对**: 明确产品定位为"工具类应用",不提供任何投资建议

### 7.4 数据隐私
- **风险**: 用户持仓数据涉及个人隐私
- **应对**: 数据仅存储在用户本地,不上传服务器
